构想：开发一个全面的个人生活/工作管理平台，可以 docker 部署，在界面上一键切换个人生活模式和工作模式，侧边栏可以开放性扩展不同的应用。个人模式包含健康管理、投资or 资产管理、阅读管理、观影剧集管理等等。工作模式包含自研开发一些工作知识库、项目管理、数据报送备份及相关管理、合同流程及生命周期管理、公司档案室及业务档案生命周期管理等等模块

一、核心技术栈
•前端架构 (Frontend)
•核心框架：Next.js (React) + TypeScript。Next.js 既支持 SPA 也能做 SSR，路由系统极其成熟。TypeScript 能让 AI 生成的代码具有严格的类型约束，大幅降低复杂业务（如合同、资产）的 Bug 率。
•构建工具：Vite (如果选择纯 React SPA) 或 Next.js 内置的 Turbopack。打包速度极快，适合频繁调试。
•状态管理：Zustand。比 Redux 轻量得多，代码样板极少，非常适合 AI 快速生成。你可以用它来管理全局的“个人/工作模式”状态切换。
•路由管理：Next.js App Router 或 React Router v6。利用嵌套路由轻松实现左侧边栏应用扩展。
•UI 组件库：Ant Design (AntD) 或 Shadcn UI。
•Ant Design 非常适合重度数据展示的工作模块（如数据报表、合同流转表单）。
•Shadcn UI 极具定制性，AI 能够很好地理解并生成现代化的界面。
•图标库：Lucide React。开源、现代，与各类 UI 库兼容极佳。
•UI 设计思路：全局采用 Context API 或 Zustand 控制 Theme（比如个人模式用暖色调/极简风，工作模式用冷色调/紧凑型高效排版）。

二、 核心模块架构
采用微内核+插件化的思路

1.工作模式 (Work Mode)
•合同流程及生命周期管理：重点聚焦于跨部门的采购协同管理与付款节点追踪。在数据表设计时，保持业务纯粹性，明确隔离此模块，不需要包含底层数据接口的测试逻辑，纯粹作为采购管理生命周期追踪。
•公司及业务档案生命周期管理：针对物理项目档案进行设计。核心字段应包含实体档案的入库位置（如密集架号）、借阅登记人、借阅期限、归还提醒以及销毁周期。在逻辑上，它应与数据报送系统完全独立。
•数据报送及备份管理：利用 Python 编写定时任务（如使用 Celery 或 APScheduler），监控自动化报送任务的状态，并抓取核心指标生成报表。
工作知识库：集成文档解析工具（如 MarkItDown 等），将日常沉淀的文档转化为 Markdown 并入库，结合向量数据库实现智能问答。

2.个人模式 (Personal Mode)
•健康管理：可以重点开发饮食与运动模块。例如建立一个食物数据库，按含糖量等指标划定“安全区”与“危险区”，辅助减脂期的饮食计划；并提供运动日程表（如足球俱乐部的训练与比赛安排）。
•投资/资产管理：支持多维度的资产录入与图表分析（前端可结合 ECharts 渲染趋势图）。
•观影与阅读：调用豆瓣或 TMDB API 自动获取元数据，实现追剧打卡与读书笔记管理。

三、开发策略
1.先定骨架，再填血肉：先让生成基础的 Docker 配置文件和带有左侧边栏、可切换双模式的空壳 UI。
2.定义严格的接口契约：在开发任何模块前，先用 TypeScript 接口 (Interfaces) 或 Pydantic 模型 (Python) 把数据结构定义清楚。
3.单模块攻坚：不要一次性让生成整个系统，而是拆分成具体的 prompt，比如：“使用 React 和 Tailwind，开发一个物理档案借阅登记的表单组件，包含借阅人、借阅时间、物理位置字段。”

四、功能特点
1. 左侧菜单栏
支持展开/折叠两种状态，默认折叠
折叠状态下鼠标悬停显示悬浮菜单，带分组分割线
展开状态下点击展开二级菜单，带旋转箭头动画
菜单项逐个加载动画

2.多语言支持
支持中文/英文切换
语言设置保存在 localStorage
菜单、标签、页面内容全部支持国际化

3.页面缓存
基于 Zustand 的状态管理
标签页切换时保持页面状态
支持动态添加/关闭标签

4.统一的背景色，Sidebar 与 Header 视觉融为一体
卡片式布局，阴影和渐变效果
响应式设计，适配不同屏幕尺寸


其他说明：
工作模式下的业务档案管理功能可以参考"dwzx-archive-openmetadata"项目，这是在 openmetadata 上去开发的业务档案管理相关功能。
